# دليل رفع mirmaz FLOW — من ملف HTML إلى موقع حقيقي

هذا الدليل ينقلك خطوة بخطوة من النسخة التجريبية إلى **موقع فعلي** فيه تسجيل دخول وصلاحiات لكل تخصص، باستخدام **Supabase** (قاعدة بيانات + مصادقة) و**Vercel** (الاستضافة).

الوقت المتوقّع: ساعة إلى ساعتين. لا تحتاج خبرة برمجية عميقة — كل خطوة محدّدة.

---

## نظرة عامة على البنية

```
المتصفح (واجهة mirmaz FLOW)
        │
        ▼
   supabase-client.js  ← طبقة الاتصال + الصلاحيات
        │
        ▼
   Supabase (سحابي)
   ├── Auth          ← تسجيل الدخول
   ├── PostgreSQL    ← كل البيانات (مهام، مراحل، فريق...)
   ├── RLS           ← الصلاحيات لكل تخصص
   └── Edge Function ← إنشاء حسابات الفريق بأمان
```

---

## الملفات في هذه الحزمة

| الملف | الوظيفة |
|------|---------|
| `01_schema.sql` | إنشاء جداول قاعدة البيانات |
| `02_permissions.sql` | نظام الصلاحiات لكل تخصص (RLS) |
| `03_seed.sql` | البيانات الأولية (الأقسام، القنوات) |
| `supabase-client.js` | طبقة الاتصال بين الواجهة وقاعدة البيانات |

| `login-screen.html` | شاشة تسجيل الدخول |

## الخطوة 1 — أنشئ مشروع Supabase (مجاني)

1. ادخل على **https://supabase.com** واضغط **Start your project**.
2. سجّل بحساب GitHub أو بريد إلكتروني.
3. اضغط **New Project**:
   - **Name:** mirmaz-flow
   - **Database Password:** اختر كلمة مرور قوية واحفظها (مهمة جداً).
   - **Region:** اختر الأقرب (مثلاً Frankfurt للعراق).
4. انتظر ~2 دقيقة حتى يجهز المشروع.

---

## الخطوة 2 — أنشئ قاعدة البيانات

1. من القائمة الجانبية اختر **SQL Editor** > **New query**.
2. افتح ملف `01_schema.sql`، انسخ محتواه كاملاً، الصقه، واضغط **Run** (أو Ctrl+Enter).
   - يجب أن ترى **Success**.
3. كرّر نفس الشيء مع `02_permissions.sql` (الصلاحiات).
4. كرّر مع `03_seed.sql` (البيانات الأولية).

الآن قاعدة البيانات جاهزة بكل الجداول والصلاحiات.

---

## الخطوة 3 — أنشئ حساب السوبر أدمن (أنت)

1. من القائمة: **Authentication** > **Users** > **Add user** > **Create new user**.
2. أدخل بريدك وكلمة مرور، وفعّل **Auto Confirm User**.
3. اضغط **Create user**، ثم انسخ الـ **User UID** الظاهر.
4. ارجع لـ **SQL Editor** وشغّل (بدّل القيم):

```sql
update public.profiles
  set role = 'super_admin', full_name = 'اسمك الكامل'
  where id = 'الصق-الـUID-هنا';
```

الآن أنت السوبر أدمن — تقدر تنشئ بقية الفريق من داخل التطبيق.

---

## الخطوة 4 — انشر دالة إنشاء المستخدمين

هذه الدالة تتيح للسوبر أدمن إنشاء حسابات الفريق بأمان.

**الطريقة الأسهل (من المتصفح):**
1. من القائمة: **Edge Functions** > **Create a function**.
2. الاسم: `create-user`.
3. الصق محتوى `create-user-function.ts` واضغط **Deploy**.

**أو عبر سطر الأوامر** (إن كان عندك Supabase CLI):
```bash
supabase functions deploy create-user
```

> الدالة تستخدم `SERVICE_ROLE_KEY` تلقائياً من بيئة Supabase — لا تحتاج إعداده يدوياً.

---

## الخطوة 5 — اربط الواجهة بقاعدة البيانات

1. افتح `supabase-client.js` وبدّل القيمتين في الأعلى:
   - من **Project Settings** > **API** انسخ:
     - **Project URL** → ضعه في `SUPABASE_URL`
     - **anon public key** → ضعه في `SUPABASE_ANON`

2. في ملف `index.html` (نسخة mirmaz FLOW)، أضف في `<head>`:

```html
<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>
<script src="supabase-client.js"></script>
```

3. انسخ محتوى `login-screen.html` والصقه داخل `<body>` (في الأعلى).

4. اربط دوال التطبيق بالبيانات الحقيقية: بدّل استدعاءات `store` المحلية بالدوال الجاهزة (`fetchTasks`, `createTask`, `saveStageEdit`...). راجع قسم "الربط بالكود" أدناه.

---

## الخطوة 6 — ارفعه على Vercel

1. ضع ملفاتك (`index.html`, `supabase-client.js`) في مجلد، وارفعه على **GitHub** (مستودع جديد).
2. ادخل **https://vercel.com** وسجّل بـ GitHub.
3. **Add New** > **Project** > اختر المستودع > **Deploy**.
4. خلال دقيقة يصير عندك رابط مثل `mirmaz-flow.vercel.app`.

**ربط دومين خاص (اختياري):** من إعدادات المشروع في Vercel > **Domains** > أضف دومينك.

## الربط بالكود (تفاصيل للمطوّر)

في الواجهة الحالية، البيانات تأتي من `store` المحلي (LocalStorage). للربط الحقيقي:

```javascript
// بدل: const store = load();
// استخدم:
let store = { team:[], sections:[], tasks:[], notifications:[], chat:null };

async function bootApp(){
  // حمّل كل شيء من Supabase بعد الدخول
  store.team    = await fetchTeam();
  store.sections= (await fetchSections()).map(s=>s.kind);
  store.tasks   = await fetchTasks();
  store.notifications = await fetchNotifications();
  store.currentUser = myId();
  render();
  // أظهر/أخفِ عناصر حسب الصلاحية:
  applyPermissions();
}

function applyPermissions(){
  // مثال: أخفِ زر "تاسك جديد" لمن لا يملك صلاحية
  document.querySelectorAll('.pill-blue.new-task')
    .forEach(b=> b.style.display = can('createTask') ? '' : 'none');
  // أخفِ الإعدادات لغير المانيجر/السوبر أدمن
  // أخفِ أزرار الموافقة لغير الإدارة... إلخ
}
```

كل عملية كتابة (إنشاء مهمة، تعديل مرحلة، موافقة) تستدعي الدالة المقابلة من `supabase-client.js` بدل تعديل `store` محلياً.

## ربط الدردشة بتيمز (المرحلة التالية)

الدردشة الحالية تعمل عبر قاعدة بيانات Supabase (آمنة ولحظية). لربطها فعلياً بـ **Microsoft Teams** للتشفير المؤسسي:

1. تحتاج **حساب Microsoft 365** للمؤسسة بصلاحiات أدمن.
2. سجّل تطبيقاً في **Azure Portal** > **App registrations**:
   - احصل على `Client ID` و `Tenant ID` و `Client Secret`.
   - أضف صلاحiات: `Chat.ReadWrite`, `ChannelMessage.Send`, `Channel.ReadBasic.All`.
3. أنشئ **Edge Function** ثانية (مثل create-user) تتواصل مع **Microsoft Graph API**:
   - عند إرسال رسالة في واجهتنا → الدالة ترسلها لقناة تيمز عبر Graph.
   - عند استقبال (Webhook من Graph) → تُخزّن وتظهر في واجهتنا.
4. واجهتنا تبقى كما هي — تيمز يصير "المحرّك المشفّر" في الخلفية.

> هذا الجزء يحتاج حساب المؤسسة جاهزاً. عند توفّره، أكتب لك دالة Graph كاملة.

## نقاط مهمة

- **الأمان:** لا تضع `SERVICE_ROLE_KEY` أبداً في كود المتصفح — فقط في Edge Functions.
- **النسخ الاحتياطي:** Supabase يعمل نسخاً تلقائية؛ يمكنك التصدير يدوياً من **Database** > **Backups**.
- **الحدود المجانية:** خطة Supabase المجانية تكفي فريقاً صغيراً (500MB قاعدة بيانات + 50K مستخدم شهرياً). للتوسّع، الترقية بسيطة.
- **التكلفة:** Supabase مجاني للبداية، Vercel مجاني للمشاريع الشخصية. الدومين الخاص ~$10/سنة (اختياري).

## الترتيب المختصر

1. أنشئ مشروع Supabase
2. شغّل ملفات SQL الثلاثة بالترتيب
3. أنشئ حساب السوبر أدمن وحدّث دوره
4. انشر دالة create-user
5. بدّل المفاتيح في supabase-client.js واربط الواجهة
6. ارفع على Vercel

بعدها يصير عندك موقع حقيقي، كل شخص يدخل بحسابه ويشوف ما يخصّه فقط.
