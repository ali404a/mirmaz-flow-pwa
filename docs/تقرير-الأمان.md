## 1. ثغرات حرجة كانت في الإصدار 1.0

### 1.1 تسريب قاعدة البيانات كاملة — حرجة

```js
const CLOUD_SYNC_URL = 'https://jsonblob.com/api/jsonBlob/019f6ab0-...';
```

رابط عام بلا مصادقة، مضمّن في HTML يقرأه أي زائر. كشف:
- كل المهام والكابشنات و44 رابط درايف
- **كل حسابات الفريق مع كلمات المرور بنص صريح** (`pass: '1234'`)
- كل المحادثات والمهام الشخصية

وقَبِل `PUT` بلا توكن — أي شخص يمسح كل شيء بأمر `curl` واحد.

**الحالة**: أُزيل بالكامل. البديل Supabase + RLS.
**إجراء مطلوب منك**: احذف الـblob، وبدّل كل كلمة مرور استُخدمت.

### 1.2 تسريب روابط الدرايف — عالية

31 مهمة حقيقية مع 44 رابط Google Drive/Canva كانت مضمّنة في الـHTML العام.

**الحالة**: نُقلت إلى `05_archive_import.sql` (خارج الواجهة).
**إجراء مطلوب منك**: راجع صلاحيات مشاركة كل مجلد.

### 1.3 تجاوز تسجيل الدخول — حرجة

زر "الوصول السريع" كان يستدعي `enterPublicDashboard()` التي تخفي شاشة الدخول
وتعرض كل البيانات — بلا أي فحص هوية.

**الحالة**: اللوحة الآن تتطلب جلسة صالحة؛ RLS يمنع القراءة بلا مصادقة.

### 1.4 مصادقة وهمية — حرجة

```js
const user = store.team.find(x => (x.user||x.id)===u && (x.pass||'1234')===p);
```

مقارنة كلمة مرور بنص صريح، في المتصفح، ضد كائن في الذاكرة. الجلسة كانت
`localStorage.setItem('mirmaz_session', user.id)` — تُزوَّر من الـconsole بسطر واحد.

**الحالة**: مصادقة Supabase حقيقية (JWT + bcrypt على الخادم).

### 1.5 تبديل الهوية — عالية

`switchUser(id)` كانت تبدّل المستخدم الحالي لأي أحد بلا مصادقة.

**الحالة**: أُزيلت. الهوية من الجلسة فقط.

### 1.6 XSS — عالية

لا دالة تهريب واحدة في الملف؛ كل شيء عبر template literals إلى `innerHTML`.
كابشن فيه `<img src=x onerror=...>` = تنفيذ فوري.

**الحالة**: `esc()` مطبّقة على كل مُدخل مستخدم. مُختبرة فعلياً هذه المرة
(probe: `<img src=x onerror=alert(1)>` يُهرَّب في `viewMaps` واللوحة).

## 2. ثغرات في طبقة RLS (لو كانت مربوطة)

| # | الثغرة | الإصلاح | مُختبرة |
|---|--------|---------|---------|
| 2.1 | `stages` update بلا `WITH CHECK` — المعيَّن ينقل المرحلة لمهمة أخرى أو يعيد تعيينها | أُضيف `WITH CHECK` يثبّت `task_id`/`kind`/`sort_order` | ✅ |
| 2.2 | `notifications` بـ `with check (true)` — انتحال هوية أي مرسِل | `actor = auth.uid()` إلزامي | ✅ |
| 2.3 | `reviewer` يمرّ في سياسة تعديل المراحل → يوافق | `can_approve()` = سوبر أدمن/إدارة فقط + حارس trigger | ✅ |
| 2.4 | `handle_new_user` يقرأ الدور من `raw_user_meta_data` (يتحكم به العميل) | لا يُقرأ إطلاقاً؛ أدنى دور دائماً، والترقية عبر Edge Function | ✅ |
| 2.5 | `prevent_self_elevation` لا يمنع المانيجر من صنع سوبر أدمن | حارس مُشدَّد: سوبر أدمن فقط يمنح/ينزع، ومانيجر لا يصنع مانيجر | ✅ |
| 2.6 | `revert_downstream_on_edit` يجبر `state='working'` حتى على مراحل الموافقة | لا يلمسها؛ ولا يجبر الحالة إن كان المحرّر ينهي المرحلة | ✅ |

```

  [PASS] مصمم يوافق على مرحلة
  [PASS] مصمم ينتحل إشعاراً باسم الكوردنيتر
  [PASS] مصمم يقرأ personal_tasks لغيره (رأى 0)
  [PASS] مصمم يعيد تعيين مرحلته لغيره
  [PASS] مدقق يوافق (ثغرة 1.0)
  [PASS] نشر قبل موافقة الإدارة

  [PASS] الإدارة توافق
  [PASS] النشر بعد الموافقة
  [PASS] سجل التدقيق last_edit_by يُضبط من الخادم
  [PASS] تعديل الكابشن يعيد الموافقة للتدقيق

=== النتيجة: 11 نجح / 0 فشل ===
```

## 5. ما زال مفتوحاً

- **93 معالج `onclick=` مضمّن** يمنع CSP صارماً. يحتاج event delegation (المرحلة 4).
- **`index.html` ملف واحد ~4000 سطر** — يحتاج تفكيكاً لوحدات.
- **الشات غير مربوط بتيمز** كما يوحي الوصف؛ هو شات Supabase عادي.
- **لا rate limiting** على محاولات الدخول (Supabase يوفّر أساسياً؛ راجع إعداداته).
- **الـEdge Function لم تُختبر حيّاً** — تحتاج مشروع Supabase فعلي.

## 6. توصيات

1. فعّل **2FA** لحساب علي (سوبر أدمن) من Supabase Auth
2. فعّل **Leaked Password Protection** في `Authentication > Policies`
3. راجع **Auth logs** دورياً
4. حساب شاشة العرض: أدنى دور، لا أدمن
5. خذ **نسخة احتياطية** قبل تشغيل `05_archive_import.sql`
